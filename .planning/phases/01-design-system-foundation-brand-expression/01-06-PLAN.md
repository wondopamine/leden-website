**Step B — Edit each file.** For every `<Button ...>` opening tag, ensure BOTH `variant=` and `size=` attributes are present. Rules:

1. **If the tag already has both props:** leave it.
2. **If `variant=` is missing:** add `variant="default"` (the component-file default) UNLESS context strongly implies a different variant (e.g., a delete CTA → `variant="destructive"`; a secondary action → `variant="outline"`). Match the visual intent already styled by surrounding `className=` overrides — if the className contains `outline-` or `border-` cues, use `variant="outline"`.
3. **If `size=` is missing:** add `size="default"` UNLESS the className contains size hints (`h-9` → `size="default"`, `h-7`/`h-8` → `size="sm"`, `h-6` → `size="xs"`, `h-11`/`h-12` → `size="lg"`, `w-9 h-9` square → `size="icon"`).
4. **Never delete an existing `className`** — even if it duplicates a variant style, leave it; a future PR can clean up. The goal of DSY-07 is prop-explicitness, not className minimization.

**Concrete example diffs:**

```diff
// src/components/order/order-content.tsx:119 (was: relied on default)
-  <Button>{tc("viewMenu")}</Button>
+  <Button variant="default" size="default">{tc("viewMenu")}</Button>

// src/components/menu/menu-content.tsx:366 (was: had className, no size/variant)
-  <Button className="flex-1 h-11 text-sm font-semibold" onClick={handleAdd}>
+  <Button variant="default" size="lg" className="flex-1 h-11 text-sm font-semibold" onClick={handleAdd}>
```

For multi-line `<Button` opens with many props, place `variant=` and `size=` on the first attribute line for legibility:

```diff
- <Button
-   className="..."
-   onClick={...}
- >
+ <Button
+   variant="default"
+   size="default"
+   className="..."
+   onClick={...}
+ >
```
